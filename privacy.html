Privacy Policy

Attendance Manager (the "App") is a local attendance management tool that communicates with ZKTeco biometric devices on your local network. This policy describes what data the App accesses and how it is handled.

Data Collected

Device Connection Credentials

  • IP address, port, and device password — used solely to connect to your ZKTeco device. Stored locally on your machine in a SQLite database. Never transmitted to any external server.

Attendance Records

  • User ID, timestamp, punch status — synced from your ZKTeco device to your local SQLite database. All data stays on your machine. Never sent to any external server.

License Validation

  • License key and machine fingerprint (SHA-256 hash of machine-uid) — sent to Keygen (api.keygen.sh) solely for license activation and validation. The fingerprint is a one-way hash and cannot be used to identify you personally.

Local Preferences

  • Locale selection (English/Arabic) and app settings — stored locally. Never transmitted.

Data Sharing

We do not sell, rent, or share your data with third parties. The only external service contacted is Keygen (api.keygen.sh) for license validation, which receives only the license key and a hashed device fingerprint.

Data Storage & Security

All attendance data, user information, and configuration are stored in a local SQLite database on your machine. No cloud backup or remote access is provided. You control the data entirely.

Third-Party Services

The App uses the following third-party services:

  • Keygen (api.keygen.sh) — license activation and validation. Keygen Privacy Policy
  • Tauri Updater — checks for app updates by fetching updates.json from GitHub Pages. No personal data is transmitted.

Data Retention & Deletion

All data is retained in the local SQLite database until you uninstall the App or manually delete it. To remove all data, uninstall the application and delete the attendance.db file from the app's data directory.

Changes to This Policy

We may update this privacy policy from time to time. Changes will be posted at this URL.
